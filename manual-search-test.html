🔍 搜索功能手动测试指南

📋 测试目标

验证主页页头搜索框的回车键搜索功能是否正常工作，确保地图优化后搜索功能不受影响。

🚀 开始测试

点击下面的按钮在新标签页打开主页进行测试：

🌐 打开主页测试 🔧 打开控制台版本

📝 详细测试步骤

步骤 1: 页面加载检查

  • 确认主页完全加载完成
  • 确认页头导航栏正常显示
  • 确认搜索框在导航栏中可见
  • 确认地图区域正常显示（本地图片）

步骤 2: 搜索框基础功能

  • 点击搜索框，确认可以获得焦点
  • 在搜索框中输入文字，确认可以正常输入
  • 确认搜索框placeholder文字显示正确
  • 确认搜索按钮（放大镜图标）可见

步骤 3: 回车键搜索测试

使用以下测试关键词逐一测试回车键搜索功能：

turbo injector actuator GT1749V Bosch diesel

对于每个关键词，执行以下操作：

  • 清空搜索框内容
  • 输入测试关键词
  • 按下 Enter 键
  • 观察是否有搜索结果显示
  • 确认搜索结果相关性
  • 记录响应时间（应该很快）

✅ 预期行为

  • 按下回车键后立即触发搜索
  • 搜索结果在页面下方正确显示
  • 搜索结果包含相关产品信息
  • 搜索响应时间 < 1秒
  • 无JavaScript错误或页面卡顿

步骤 4: 搜索按钮测试

  • 在搜索框中输入关键词
  • 点击搜索按钮（放大镜图标）
  • 确认搜索功能正常触发
  • 对比回车键和按钮点击的效果是否一致

步骤 5: 边界情况测试

  • 测试空搜索（不输入内容直接回车）
  • 测试特殊字符搜索
  • 测试很长的搜索词
  • 测试中文搜索词
  • 测试连续快速搜索

🐛 故障排查

⚠️ 如果搜索功能不工作

请按以下步骤排查：

  1. 检查控制台错误：按F12打开开发者工具，查看Console标签页是否有错误信息
  2. 检查网络请求：在Network标签页查看是否有失败的请求
  3. 检查元素存在：确认搜索框ID为"searchInput"
  4. 检查事件绑定：在控制台输入以下代码检查：
// 检查搜索框是否存在
console.log('搜索框:', document.getElementById('searchInput'));

// 检查搜索函数是否存在
console.log('搜索函数:', typeof performSearch);

// 手动触发搜索测试
document.getElementById('searchInput').value = 'turbo';
performSearch();

🔧 常见问题解决

  • 搜索框不响应：可能是事件监听器未正确绑定，检查main.js是否正常加载
  • 搜索结果不显示：可能是产品数据未加载，检查网络请求
  • 页面卡顿：可能是JavaScript错误，检查控制台
  • 搜索结果错误：可能是搜索逻辑问题，检查performSearch函数
